    在GitHub上手动deploy Mkdocs

    1. 根据mkdocs的官方指引，创建好本地文件（e.g., 储存在一个叫notebook的文件夹里）
    2. 在GitHub上创建一个新的repo，例如 https://github.com/username/reponame
    3. 在notebook的目录下输入以下指令 git init # build the local repo git add . # track all the local files git commit -m "commit message" git remote add origin https://github.com/username/reponame.git # add the remote repo referenced by the url, and rename it "origin" locally git push -u origin main # push the change from local branch "main" to remote branch "origin"
    4. 输入mkdocs gh-deploy，GitHub自动将页面deploy后即可查看网页

    在第2步中，如注释所写，最后一条指令是将本地的main分支的内容push到远程的origin分支中。其中远程分支的名字origin在git remote add ...中指定，而本地分支可以用git branch来查看，否则可能会报错。

    提醒后续更新

    根据mkdocs-material的解释，此方法需要在进行本地修改后重新进行deploy才能看到更新。
